# React Starter Project Explanation

This document explains every folder and file inside the `my-react-app` starter project generated by Vite for React. If you are new to React, this guide will help you understand how the app is structured and how the pieces connect.

---

## Project root: `my-react-app/`

This is the top-level folder for your React project. It contains all project configuration, source code, dependencies, and build settings.

### `.gitignore`
- Purpose: tells Git which files and folders should not be tracked by version control.
- Why it exists: some files are generated automatically or specific to your machine, and they should not be shared in the repository.
- Key entries in this starter:
  - `node_modules/` — dependency packages installed by npm.
  - `dist/` and `dist-ssr/` — build output folders created when you compile the app.
  - local editor settings files like `.vscode/` and `.idea`.

### `README.md`
- Purpose: documentation for the project.
- What it contains: a short explanation of the template, the tools used (React + Vite), and notes about React Compiler and ESLint.
- Why it is useful: it helps anyone opening the project understand the starter template and where to look for official plugin/documentation links.

### `eslint.config.js`
- Purpose: configuration file for ESLint, a tool that finds problems in JavaScript code.
- Why it exists: to enforce coding style and catch mistakes before they become bugs.
- In this starter: it provides linting rules suitable for React code, including plugin support for React hooks and React Refresh.

### `index.html`
- Purpose: the main HTML file loaded by the browser when the app starts.
- Key parts:
  - `<!doctype html>`: declares this as an HTML5 page.
  - `<div id="root"></div>`: the placeholder where React will render the app.
  - `<script type="module" src="/src/main.jsx"></script>`: loads the React app entry file.
- Why it matters: React does not replace the entire HTML page; it only renders inside the `#root` container.

### `package.json`
- Purpose: metadata and dependency definitions for the project.
- Important sections:
  - `name`: project name.
  - `private`: prevents accidental package publishing.
  - `version`: app version.
  - `type`: `module` means JavaScript files use ES modules by default.
  - `scripts`: commands you can run with npm.
    - `npm run dev` starts the local development server.
    - `npm run build` compiles the app for production.
    - `npm run lint` checks code with ESLint.
    - `npm run preview` runs the production build locally.
  - `dependencies`: packages needed to run the app.
    - `react` and `react-dom` are required to build and render React components.
  - `devDependencies`: packages needed only for development.
    - `vite` is the build tool and dev server.
    - `@vitejs/plugin-react` integrates React with Vite.
    - ESLint packages handle code linting.
    - `@types/react` and `@types/react-dom` are type definitions used by TypeScript-aware tooling, but they don’t affect runtime.

### `package-lock.json`
- Purpose: locks the exact version of every installed dependency.
- Why it exists: to make sure all developers and deployment environments install the same dependency versions.
- Important note: you usually do not edit this file manually.

### `vite.config.js`
- Purpose: Vite configuration.
- What it does:
  - imports `defineConfig` from Vite.
  - imports `@vitejs/plugin-react` to enable React support.
  - exports a default configuration object with `plugins: [react()]`.
- Why it matters: Vite needs this file to know how to process the React app when running or building.

### `node_modules/`
- Purpose: contains all installed dependency packages.
- Why it exists: npm downloads and stores the code for React, Vite, ESLint, and every plugin here.
- Important note: this folder is generated automatically and usually should not be edited directly.

### `public/`
- Purpose: static assets served directly by Vite.
- In this project:
  - `favicon.svg` — the icon that appears in the browser tab.
  - `icons.svg` — a collection of SVG icons used by the starter app.
- How it works: files in `public/` are copied as-is to the final build and can be referenced in HTML and React code.

## Source folder: `src/`

This is where the actual application code lives.

### `src/main.jsx`
- Purpose: the app entry file.
- What it does:
  - imports `StrictMode` from React.
  - imports `createRoot` from `react-dom/client`.
  - imports `./index.css` for global styles.
  - imports the `App` component from `./App.jsx`.
  - selects the HTML element with `id="root"`.
  - tells React to render `<App />` inside the `root` element.
- Why it is important: it connects the React code to the web page.

### `src/App.jsx`
- Purpose: the main React component of the starter app.
- Key parts:
  - `import { useState } from 'react'`:
    - `useState` is a React hook that allows the component to remember state and update it.
  - `import reactLogo from './assets/react.svg'` and `viteLogo from './assets/vite.svg'`:
    - these imports bring image files into the component so React can display them.
  - `import heroImg from './assets/hero.png'`:
    - another image used in the page design.
  - `import './App.css'`:
    - loads component-specific CSS styling.

- The `App` function:
  - defines a state variable: `const [count, setCount] = useState(0)`.
    - `count` is the current value.
    - `setCount` is the function used to change it.
    - `useState(0)` sets the initial value to `0`.
  - returns JSX:
    - A fragment `<>...</>` contains multiple sections.
    - `<section id="center">...</section>` is the main content area.
    - It renders the images, a heading, a paragraph, and a button.
    - The button has an `onClick` handler: `onClick={() => setCount((count) => count + 1)}`.
      - every time the button is clicked, the `count` increases by 1.
      - React then re-renders the component and updates the displayed value.
    - The text `Count is {count}` shows the current number.
  - uses inline markup and class names to connect to CSS styling.

- `export default App`:
  - exports the component so `main.jsx` can import and render it.

- Why it matters: this file is the first real React code you modify when building your app.

### `src/App.css`
- Purpose: styling specifically for the `App` component.
- What it contains:
  - styles for the `.counter` button, including hover and focus behavior.
  - layout styles for `.hero`, which positions the logos and hero image.
  - responsive design rules with `@media (max-width: 1024px)` for smaller screens.
  - styles for `#center`, `#next-steps`, `#docs`, `#social`, `.ticks`, and `#spacer`.
- Why it is important: it defines how the app looks and how the page responds to different screen sizes.

### `src/index.css`
- Purpose: global styles for the entire app.
- What it contains:
  - CSS variables defined under `:root`, such as `--text`, `--bg`, `--accent`, and `--shadow`.
  - default font settings and color scheme support.
  - a dark mode theme using `@media (prefers-color-scheme: dark)`.
  - global page styles:
    - `body { margin: 0; }` removes default browser spacing.
    - `#root` is styled to center the app and make it fill the viewport.
    - heading styles for `h1`, `h2`, `p`, and `code`.
- Why it matters: these styles apply everywhere, not just inside a single component.

### `src/assets/`
- Purpose: stores image assets used by the starter page.
- In this starter, it contains:
  - `react.svg` — the React logo.
  - `vite.svg` — the Vite logo.
  - `hero.png` — a decorative image used in the starter page.
- Why it matters: React allows you to import these image files directly into component code.

## How the app starts and runs

1. You run `npm install` once to download dependencies into `node_modules/`.
2. You run `npm run dev` to start the Vite development server.
3. Vite serves `index.html` in the browser.
4. `index.html` loads `src/main.jsx`.
5. `src/main.jsx` renders the `App` component inside `<div id="root"></div>`.
6. `App.jsx` returns JSX, React builds the page structure, and the CSS files make it look styled.

## Why this starter code is useful for beginners

- It shows the standard React application entry point (`main.jsx`).
- It demonstrates a simple functional component (`App`) with state.
- It includes both global styling (`index.css`) and component styling (`App.css`).
- It uses Vite, which gives fast reloads and modern build behavior.
- It uses `package.json` scripts that are easy to run.

## Quick glossary of key terms

- **React component**: a JavaScript function or class that returns UI.
- **JSX**: the HTML-like code inside React components.
- **State**: data that a component remembers and can change.
- **Hook**: a special React function such as `useState`.
- **Vite**: a fast build tool and local development server.
- **ESLint**: a code quality tool that checks for errors and style problems.
- **`public/` folder**: static files served unchanged.
- **`src/` folder**: the app source code that gets processed and bundled.

## What you can change first

- Open `src/App.jsx` and change the text in `<h1>`.
- Change the button label and see it update automatically.
- Modify `src/index.css` colors to see how global styles change the page.
- Add a new paragraph inside `App.jsx` to learn how JSX works.

## Summary

This starter project is a minimal React application built with Vite. The most important files are:
- `src/main.jsx` — connects React to the browser.
- `src/App.jsx` — contains the app UI and React logic.
- `src/index.css` and `src/App.css` — control styling.
- `package.json` — defines how to run, build, and lint the app.

In the browser, React renders the `App` component into the `root` element inside `index.html`, and Vite makes development fast.
